# multi-platform-game-design-system
Software design for a distributed, web-based game application supporting multiple teams and players. Demonstrates system architecture, domain modeling, memory management, and scalable Linux-based platform recommendations. Completed as part of the CS-230 Operating Platforms Final Project.

This project was completed as part of the CS-230 Operating Platforms course and focuses on designing a software solution for a fictional client, The Gaming Room. The client requested an expansion of their Android-based game, Draw It or Lose It, into a web-based, multi-platform application capable of supporting multiple games, teams, and players. The design emphasizes scalability, performance, and security while operating in a distributed environment. In developing this software design document, I focused on organizing system requirements clearly, applying object-oriented principles, and using design patterns such as Singleton to ensure efficient memory management. Working through the design process prior to implementation helped establish a strong foundation and improved overall clarity in how system components interact.

If I were to revise this project, I would enhance the system architecture section by incorporating more detailed diagrams to better represent communication between distributed components. Throughout the process, I prioritized interpreting user needs by focusing on accessibility, scalability, and usability, as these are critical to the success of a multi-platform application. My approach involved breaking the system into core components such as Game, Team, and Player, and structuring their relationships to reflect real-world functionality. Moving forward, I would continue using structured design practices, visual modeling, and established architectural patterns to develop efficient, maintainable, and scalable software systems. Now that I have completed the course and have a stronger understanding of the full software design process, I believe each section of the document could be expanded with greater depth and technical detail.
